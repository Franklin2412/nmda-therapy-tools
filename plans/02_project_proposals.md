# Project Proposals: Web-Based Therapeutic Tools for NMDA Encephalitis Recovery

## Overview

This document outlines potential web-based applications designed to support children recovering from autoimmune NMDA encephalitis. Each project focuses on specific rehabilitation needs identified in clinical research and can be deployed on GitHub Pages for easy demonstration and accessibility.

---

## Project 1: Memory Garden üå±

### Purpose
An engaging memory training application that helps children rebuild working memory, visual memory, and sequential processing through garden-themed games.

### Target Skills
- **Visual Memory**: Remembering patterns, locations, objects
- **Sequential Memory**: Recalling order of events
- **Working Memory**: Holding and manipulating information
- **Attention**: Sustained focus on tasks

### Features

#### Level 1: Flower Memory Match
- Classic memory matching game with colorful flowers
- Adjustable difficulty (4, 8, 12, 16 cards)
- Tracks accuracy and time
- Celebratory animations for matches

#### Level 2: Garden Sequence
- Child watches flowers bloom in a specific sequence
- Must recreate the blooming sequence
- Progressively longer sequences (3 ‚Üí 8 items)
- Audio-visual cues for reinforcement

#### Level 3: Treasure Hunt
- Remember locations of hidden items in a garden scene
- Items disappear after viewing period
- Must click correct locations from memory
- Spatial memory training

#### Level 4: Story Seeds
- Short illustrated stories presented step-by-step
- Child must recall story sequence
- Drag-and-drop story cards in correct order
- Builds narrative memory

### Technical Implementation
- **Frontend**: HTML5, CSS3, JavaScript (vanilla)
- **Graphics**: CSS animations, SVG illustrations
- **Storage**: LocalStorage for progress tracking
- **Accessibility**: Keyboard navigation, high contrast mode, adjustable text size

### Progress Tracking
- Session history (dates, scores)
- Difficulty level progression
- Time spent per activity
- Success rate trends
- Downloadable progress report (CSV/PDF) for parents/doctors

## Project 2: SpeakEasy Studio üé§

### Purpose

#### Module 2: Sentence Builder
- Drag-and-drop word cards to form sentences
- Picture prompts to encourage description
- Progressive complexity (3 words ‚Üí full sentences)
- Correct grammar celebration

#### Module 3: Story Teller
- Picture sequences for storytelling
- Child describes what's happening (text or voice)
- Prompts for who, what, where, when, why
- Builds narrative skills

#### Module 4: Word Hunt
- Themed vocabulary games (animals, food, colors, actions)
- Picture-word matching
- Fill-in-the-blank sentences
- Word categorization activities

#### Module 5: Listen & Do
- Follow simple to complex instructions
- "Simon Says" style activities
- Multi-step direction practice
- Auditory processing training

### Technical Implementation
- **Frontend**: HTML5, CSS3, JavaScript
- **Audio**: Web Audio API for recording/playback
- **Graphics**: Animated SVG characters, colorful UI
- **Storage**: LocalStorage for vocabulary progress

### Progress Tracking
- Words mastered
- Sentence complexity achieved

- Recording history
- Downloadable practice log

### Purpose

- Rapidly changing stimuli requiring quick responses

#### Game 4: Puzzle Path
- Maze navigation with increasing complexity
- Planning optimal routes
- Multi-step problem solving
- Spatial reasoning

#### Game 5: Number Ninjas
- Basic math games (appropriate to age/ability)
- Number sequencing
- Simple addition/subtraction in context
- Cognitive flexibility with numbers

- **Frontend**: HTML5, Canvas API for graphics, CSS3, JavaScript
- **Animation**: Smooth transitions and rewards
- **Difficulty Adjustment**: Adaptive difficulty based on performance
- **Storage**: LocalStorage for game history

- Games completed by difficulty

## Cross-Project Features

### Universal Design Elements

#### 1. Child-Friendly Interface
- **Bright, engaging colors** with high contrast options
- **Large buttons and clear icons**
- **Minimal text, maximum visuals**
- **Cheerful animations and feedback**
- **No frustrating time limits** (unless part of therapeutic goal)

#### 2. Adaptive Difficulty
- **Automatic adjustment** based on performance
- **Manual override** for parents/therapists
- **"Just right" challenge** level to maintain engagement
- **No punitive feedback** for mistakes

#### 3. Positive Reinforcement
- **Celebratory animations** for successes
- **Encouraging messages** throughout
- **Achievement badges and stars**
- **"You're doing great!" feedback**
- **No negative language** or "failure" states

#### 4. Accessibility
- **Keyboard navigation** throughout
- **Screen reader compatibility**
- **Adjustable text sizes**
- **High contrast mode**
- **Colorblind-friendly palettes**
- **Subtitle/caption options** for audio

#### 5. Parent/Therapist Dashboard
- **Separate access mode** with password/PIN
- **View all progress data**
- **Customize difficulty levels**
- **Enable/disable specific activities**
- **Export reports** (PDF, CSV)
- **Set therapeutic goals**

#### 6. Privacy & Safety
- **No user accounts required** (everything local)
- **No data transmission to servers**
- **No advertisements**
- **No external links** (safe browsing)
- **COPPA compliant** design
- **Parent control over all features**

#### 7. Offline Capability
- **Progressive Web App (PWA)** architecture
- **Works without internet** after initial load
- **Local data storage**
- **Install to device** option

- **Service Workers**: Offline functionality (PWA)
- **Web Speech API**: Voice recognition for speech exercises
- **MediaDevices API**: Webcam for optional movement tracking

### Deployment
- **GitHub Pages**: Free hosting
- **Responsive Design**: Works on tablets, computers
- **No server required**: Purely client-side applications
- **Easy updates**: Push to repository to update

## Medical Validation Plan

### Steps for Clinical Feedback

1. **Create Working Prototypes**
   - Build MVP (Minimum Viable Product) of 1-2 projects
   - Ensure core features work smoothly
   - Test with family members first

2. **Prepare Demo Package**
   - Live demos on GitHub Pages
   - Video walkthrough of features
   - Written documentation of therapeutic goals
   - This planning document

3. **Consult with Medical Team**
   - Present to treating physicians
   - Get feedback from speech therapists
   - Consult with occupational therapists
   - Involve neuropsychologists

4. **Iterate Based on Feedback**
   - Modify activities based on clinical input
   - Adjust difficulty levels
   - Add/remove features as recommended
   - Refine therapeutic goals

5. **Pilot Testing**
   - Test with small group of patients (with approval)
   - Gather usage data and feedback
   - Assess engagement and effectiveness
   - Make final adjustments

6. **Documentation & Sharing**
   - Create user guides for parents
   - Therapist guides with therapeutic goals
   - Open-source on GitHub for wider access
   - Share with encephalitis support communities
